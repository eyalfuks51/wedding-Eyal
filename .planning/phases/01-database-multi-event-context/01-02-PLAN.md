   interface EventContextValue {

   import { useAuth } from '@/contexts/AuthContext';

4. **Implement localStorage persistence with stale fallback:**
   - Constant: `const STORAGE_KEY = 'currentEventId';`
   - `resolveCurrentEvent(events)`: check localStorage for stored ID, find match in events array. If match found, return it. If not found (stale), `localStorage.removeItem(STORAGE_KEY)` and return `events[0]`. If events empty, return null.

5. **Implement client-side sorting:**
   ```typescript
   function sortEvents(events: EventData[]): EventData[] {
     return [...events].sort((a, b) => (b.event_date || '').localeCompare(a.event_date || ''));
   }
   ```
   Per user decision: sort by `event_date DESC`, fallback to empty string for nulls.

6. **EventProvider implementation:**
   - Consume `useAuth()` to get `user` and `isSuperAdmin`.
   - Guard fetch on `user?.id` being non-null (prevents race condition — Pitfall 3 from research).
   - Choose fetch function: `isSuperAdmin ? fetchAllEvents : fetchEventsForUser`.
   - After fetch: sort events, resolve current event via localStorage, set state.
   - Cancellation pattern with `let cancelled = false` (same as current code).
   - Dependencies: `[user?.id, isSuperAdmin, tick]`.

7. **switchEvent implementation:**
   - Find event by ID in events array.
   - If found: `setCurrentEvent(match)` + `localStorage.setItem(STORAGE_KEY, id)`.
   - If not found: no-op (defensive).
   - Wrap in `useCallback` with `[events]` dependency.
   - Per user decision: NO navigation, NO page reload — soft context refresh only.

8. **Provider value:**
   ```typescript
   { events, currentEvent, isActive: currentEvent?.status === 'active', isLoading, switchEvent, refetch: () => setTick(t => t + 1) }
   ```

9. **Keep `useEventContext` hook** with same throw behavior.
