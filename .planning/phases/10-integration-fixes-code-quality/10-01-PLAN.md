**AuthContext.tsx (INT-01):**
Replace the `onAuthStateChange` callback (line 42-44) with an async handler that re-queries `is_super_admin` on token refresh:

```typescript
const { data: { subscription } } = supabase!.auth.onAuthStateChange(
  async (_event, newSession) => {
    if (cancelled) return;
    setSession(newSession);
    const uid = newSession?.user?.id;
    if (uid) {
      const { data } = await supabase!
        .from('users')
        .select('is_super_admin')
        .eq('id', uid)
        .single();
      if (!cancelled) setIsSuperAdmin(data?.is_super_admin ?? false);
    } else {
      if (!cancelled) setIsSuperAdmin(false);
    }
  }
);
```

The `cancelled` flag from the existing `useEffect` closure is already in scope. No other changes needed.

**ProtectedRoute.tsx (INT-02):**
In `ProtectedRouteInner`, when `events.length === 0` but `localStorage.getItem('currentEventId')` exists, call `refetch()` with retry logic before redirecting to `/onboarding`.

Add `refetch` to the destructured `useEventContext()` call. Add state: `const [retryCount, setRetryCount] = useState(0)`. Add a `useEffect` that:
1. Checks if `!authLoading && !eventLoading && events.length === 0 && localStorage.getItem('currentEventId')`
2. If `retryCount < 3`, calls `refetch()` after 500ms timeout, increments retryCount
3. If `retryCount >= 3`, falls through (allows the existing Navigate to /onboarding to render)

Import `useState` and `useEffect` from React. Keep the existing guard logic as-is but add a condition: if `events.length === 0` AND `retryCount < 3` AND `localStorage.getItem('currentEventId')`, render `<Spinner />` instead of `<Navigate to="/onboarding">`.

Anti-patterns: Do NOT use a bare setTimeout as the primary fix. Use proper retry with state tracking and cancellation. Do NOT make the retry blocking or add a global loading delay.
