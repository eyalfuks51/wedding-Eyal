Also fix the slug generation. Currently in OnboardingPage.tsx line 27-29 the slug is generated with a regex that strips Hebrew chars. Move slug generation into a helper function in supabase.js (or keep inline -- either is fine). The fix:

```javascript
// Replace the current slug generation logic:
// OLD: base = name1-and-name2 lowercase, strip non [a-z0-9-], append Date.now()
// NEW: Keep Hebrew chars in URL, add 6-char random suffix for uniqueness
function generateSlug(p1, p2) {
  const base = `${p1}-and-${p2}`
    .toLowerCase()
    .replace(/\s+/g, '-')
    .replace(/[^a-z0-9\u0590-\u05ff-]/g, '') // Keep Hebrew + Latin + digits + hyphens
    .replace(/-+/g, '-')     // Collapse multiple hyphens
    .replace(/^-|-$/g, '')   // Trim leading/trailing hyphens
    .slice(0, 40);
  const suffix = Math.random().toString(36).slice(2, 8);
  return base ? `${base}-${suffix}` : `event-${suffix}`;
}
```

Export `generateSlug` so OnboardingPage.tsx can call it. The function does NOT need to be async or hit the DB -- the UNIQUE constraint on `events.slug` handles collision (extremely rare with 6 random chars).

Add a lightweight `RequireAuth` wrapper component. This is simpler than `ProtectedRoute` because onboarding does NOT need EventProvider -- it just needs to ensure the user is logged in.

Option A (inline in App.jsx): Add a small component at the top of the file:

```jsx
import { Navigate } from 'react-router-dom';
import { useAuth } from './contexts/AuthContext';

function RequireAuth({ children }) {
  const { user, loading } = useAuth();
  if (loading) return (
    <div className="min-h-screen flex items-center justify-center">
      <div className="animate-spin h-8 w-8 border-2 border-violet-600 border-t-transparent rounded-full" />
    </div>
  );
  if (!user) return <Navigate to="/login" replace />;
  return children;
}
```

Then wrap the onboarding route:
```jsx
<Route path="/onboarding" element={<RequireAuth><OnboardingPage /></RequireAuth>} />
```

The comment above the route currently says "Auth-required, no event needed" but has no actual guard -- this fixes that.
