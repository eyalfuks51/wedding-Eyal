1. **Line 285** (`(currentEvent as any).content_config`): Replace with `currentEvent.content_config`. The component already null-guards `currentEvent` above. **LivePreview.tsx — widen event prop type:**

Update the `LivePreviewProps` interface so `event_date` accepts `string | null`:
```typescript
interface LivePreviewProps {
  event: { id: string; slug: string; template_id: string; event_date: string | null };
  config: Record<string, any>;
  width?: number;
}
```
The iframe URL uses `event.slug`, not `event_date`, so null date is harmless.

**POLISH-02: EventContext.tsx — gate fetch on authLoading:**

In `EventContext.tsx`:
1. Destructure `loading` from `useAuth()` with alias: `const { user, isSuperAdmin, loading: authLoading } = useAuth();`
2. Add early return at the top of the `useEffect` body: `if (authLoading) return;`
3. Add `authLoading` to the dependency array: `[user?.id, isSuperAdmin, authLoading, tick]`

This ensures the effect fires only once after both `user` and `isSuperAdmin` are settled, eliminating the double-fetch for super admins.
