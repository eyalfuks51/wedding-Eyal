## Tech Debt Summary (6 items)

### High Risk
1. **`updated_at` column reference** (RSVP-02, RSVP-03) — `sync_arrival_to_invitation` and `link_permit_to_invitation` triggers execute `UPDATE invitations SET updated_at = now()` but no tracked migration adds this column to `invitations`. May rely on pre-existing schema. If absent, RSVP submissions and unmatched permit resolution will fail at the database level.

### Medium Risk
2. **Three separate EventProvider instances** (AUTH-02, NAV-01) — `App.jsx` wraps each `/dashboard/*` route in its own `ProtectedRoute`→`EventProvider`, causing a Supabase fetch on every tab navigation and no cross-tab state sharing. Functional correctness preserved via localStorage.

### Low Risk
3. **`canAccessSettings` dead export** (GATE-01) — `useFeatureAccess` defines `canAccessSettings: true as const` but no component reads it.
4. **`isValidPhone` not used at runtime** (TEST-03, GATE-03) — Phone validation exists in tests and Excel import but not in manual Add Guest or EditGuestSheet flows.
5. **E2E test hardcoded slug** (E2E-02) — `rsvp.spec.ts` targets `hagit-and-itai` only; won't cover new multi-tenant events.
6. **ROADMAP Plan 11-01 checkbox** (POLISH-03) — Shows `[ ]` despite phase completion. Minor bookkeeping.

**Total: 6 items — zero runtime blockers, zero requirement failures**
